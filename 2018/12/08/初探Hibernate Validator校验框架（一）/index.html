初探Hibernate Validator校验框架（一）:声明bean约束

发表于 2018-12-08 | 分类于 java | | 阅读次数:

简介

校验数据是一个从表现层贯穿到持久层的所有应用程序层都会出现的常见任务。通常在每个应用程序层中实现相同的验证逻辑的方法非常耗时而且很容易出错。为了避免重复这些验证，开发人员经常将验证逻辑直接捆绑到域模型中(Domain Model)，使用验证代码(实际上是关于类本身的元数据)来统一管理校验数据的功能。

Hibernate-Validator就是使用上面原理的一个很好的校验框架，以下简称为H-V。它为实体和方法校验定义了元数据模型和API。我们只需要在我们的WEB项目中引入依赖就可以完成实体和方法的校验了。H-V可以使用注解进行配置。

今天，我们先来学习一下如何声明一个bean约束。

声明bean约束

Bean验证中的约束是通过Java注解表示的。我们可以通过这些注解来增强对象的实际性能。

bean约束一共有四种类型：

  • 字段约束
  • 属性约束
  • 容器元素约束
  • 类约束

此外，我们还将介绍：

  • 约束继承
  • 对象图（即级联约束）

使用字段级的约束时，字段的访问策略用于访问要验证的值。H-V的验证引擎会直接访问实例变量的值。（不调用属性约束访问器的方法访问，即使有属性约束访问器）

字段约束可以应用于任何访问类型的字段（public、private等等）。但是它不支持对静态字段的约束，即定义为static的字段。

如果你需要验证字节码增强对象时，你需要使用属性级约束，而不是字段级约束。因为字节码增强库无法通过反射来确定字段的访问。属性级约束也就是我们下面将要讨论的。

注意：建议在一个类中只选择 字段注解和属性注解中的一种方式。因为这样会导致字段被验证两次。

Bean Validation API不仅允许验证单个类实例，还允许完成对象图（级联验证）。为此，只需要在引用了其他对象的字段或者属性上添加一个@Valid注解即可。实例如下：

如果Car的实例被验证，那么它引用的Person对象也会被验证，因为driver字段添加了@Valid注解。 因此如果Person实例的name字段是为空，那么对Car的验证将会失败。

当然，容器元素约束也是支持对象图（级联验证）的。这就意味着一个容器中的任何类型字段都可以加上@Valid注解，使得当父类被验证时它的每一个子类容器元素都会被验证。
